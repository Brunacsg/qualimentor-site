Tipos de teste de performance

Load Testing

Avalia comportamento sob carga esperada normal.

Stress Testing

Testa limites do sistema sob carga extrema.

Spike Testing

Simula picos repentinos de carga.

Volume Testing

Testa com grandes volumes de dados.

Endurance Testing

Testa estabilidade por longos períodos.

Scalability Testing

Avalia capacidade de escalar recursos.

Exemplo: Stress Test Scenario

Teste de Estresse com k6

import http from 'k6/http';
import { check, sleep } from 'k6';

export let options = {
  stages: [
    { duration: '2m', target: 100 }, // Ramp up to 100 users
    { duration: '5m', target: 100 }, // Stay at 100 users
    { duration: '2m', target: 200 }, // Ramp up to 200 users
    { duration: '5m', target: 200 }, // Stay at 200 users
    { duration: '2m', target: 0 },   // Ramp down to 0 users
  ],
};

export default function () {
  let response = http.get('https://api.example.com/users');
  check(response, { 'status is 200': (r) => r.status === 200 });
  sleep(1);
}

Objetivo: Encontrar o ponto de quebra do sistema

Métricas críticas: Response time degradation, error rate increase

Ferramentas de performance

JMeter

Open source, protocolos diversos, relatórios detalhados.

LoadRunner

Comercial, enterprise-grade, suporte a protocolos proprietários.

k6

Moderno, JavaScript, cloud-native, fácil integração CI/CD.

Gatling

Scala-based, alta performance, DSL expressivo.

Monitoramento em produção

  • Application Performance Monitoring (APM): New Relic, DataDog, Dynatrace.
  • Infrastructure Monitoring: Prometheus, Grafana, Nagios.
  • Real User Monitoring (RUM): Google Analytics, Pingdom.
  • Log Analysis: ELK Stack, Splunk.

Otimização de performance

Frontend

Minificação, compressão, CDN, lazy loading, caching.

Backend

Database optimization, connection pooling, async processing.

Infrastructure

Load balancing, auto-scaling, caching layers (Redis, Memcached).

Database

Indexing, query optimization, read replicas, sharding.
