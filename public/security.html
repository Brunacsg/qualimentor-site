OWASP Top 10

Injection

SQL Injection, Command Injection, LDAP Injection.

Broken Authentication

Sessões fracas, credential stuffing, session fixation.

Sensitive Data Exposure

Dados não criptografados, weak encryption.

XML External Entities

XXE attacks em parsers XML.

Broken Access Control

IDOR, privilege escalation, insecure direct object references.

Security Misconfiguration

Configurações padrão, headers inseguros, verbose errors.

Cross-Site Scripting

XSS reflected, stored, DOM-based.

Insecure Deserialization

Object injection, remote code execution.

Vulnerable Components

Libraries desatualizadas com vulnerabilidades conhecidas.

Insufficient Logging

Falta de logs para detecção de ataques.

Ferramentas de segurança

  • Burp Suite: Web vulnerability scanner e proxy.
  • OWASP ZAP: Open source web app scanner.
  • Nessus: Vulnerability scanner comercial.
  • SQLMap: Automated SQL injection tool.
  • Nmap: Network discovery and security auditing.
  • Wireshark: Network protocol analyzer.

Testes de segurança comuns

Penetration Testing

Simulação de ataques reais ao sistema.

Análise estática do código fonte.

Vulnerability Assessment

Identificação de vulnerabilidades conhecidas.

Compliance Testing

Verificação contra padrões (GDPR, PCI DSS, etc.).

Exemplo prático: SQL Injection

Ataque SQL Injection

Input malicioso: ' OR '1'='1

Query original: SELECT * FROM users WHERE email = '$email' AND password = '$password'

Query resultante: SELECT * FROM users WHERE email = '' OR '1'='1' AND password = ''

Resultado: Login sem credenciais válidas

Prevenção: Usar prepared statements ou ORMs

Exemplo: XSS Attack

Cross-Site Scripting

Input malicioso: <script>alert('XSS')</script>

Resultado: Script executado no browser do usuário

Teste: Inserir payloads em campos de input e verificar se são executados

Prevenção: Sanitizar inputs, usar Content Security Policy (CSP)

Exemplo: Teste com OWASP ZAP

Automated Scan com ZAP

1. Configurar contexto: Definir URLs da aplicação

2. Spider: Mapear todas as páginas e endpoints

3. Active Scan: Executar ataques automatizados

4. Relatório: Vulnerabilidades encontradas por severidade

Exemplo de alerta: SQL Injection em /api/users?id=1'

Exemplo: Penetration Test Checklist

Checklist de Pentest

  • ✅ Reconhecimento (nmap, dns enumeration)
  • ✅ Scanning (port scanning, service detection)
  • ✅ Gaining Access (exploit vulnerabilities)
  • ✅ Maintaining Access (backdoors, persistence)
  • ✅ Covering Tracks (log cleaning)
  • ✅ Reporting (findings, risks, recommendations)
