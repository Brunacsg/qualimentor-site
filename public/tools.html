Exemplo prático: Usando Postman

Criando coleção de testes de API

1. Criar Collection: "API Usuários"

2. Adicionar Request: POST /api/users

3. Body (JSON):

{
  "name": "João Silva",
  "email": "joao@teste.com",
  "password": "senha123"
}

4. Tests (JavaScript):

pm.test("Status code is 201", function () {
    pm.response.to.have.status(201);
});

pm.test("Response has user id", function () {
    var jsonData = pm.response.json();
    pm.expect(jsonData).to.have.property('id');
});

Exemplo: Query SQL para testes de dados

Verificar integridade de dados

-- Verificar usuários duplicados
SELECT email, COUNT(*) as count
FROM users
GROUP BY email
HAVING COUNT(*) > 1;

-- Verificar pedidos sem usuário
SELECT o.id, o.user_id
FROM orders o
LEFT JOIN users u ON o.user_id = u.id
WHERE u.id IS NULL;

-- Verificar produtos com preço inválido
SELECT id, name, price
FROM products
WHERE price <= 0 OR price > 99999.99;

Exemplo: Configuração Jenkins Pipeline

Jenkinsfile para QA

pipeline {
    agent any
    stages {
        stage('Test') {
            steps {
                sh 'npm install'
                sh 'npm run test:unit'
                sh 'npm run test:e2e'
            }
        }
        stage('Security Scan') {
            steps {
                sh 'npm run security-scan'
            }
        }
        stage('Performance Test') {
            steps {
                sh 'npm run performance-test'
            }
        }
    }
    post {
        always {
            publishTestResults()
            sendNotifications()
        }
    }
}
