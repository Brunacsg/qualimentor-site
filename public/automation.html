Por que automatizar testes?

  • Redução significativa de tempo em testes repetitivos e regressão.
  • Maior confiabilidade e consistência na execução.
  • Execução contínua em pipelines de CI/CD.
  • Validação rápida de integrações críticas.
  • Liberação de QA para testes exploratórios e de valor agregado.
  • Feedback mais rápido para desenvolvedores.
  • Redução de custos a longo prazo.

Quando NÃO automatizar

  • Testes que serão executados apenas uma vez.
  • Testes com requisitos voláteis e mudanças frequentes.
  • Testes exploratórios que dependem de criatividade humana.
  • Testes de usabilidade que envolvem opinião subjetiva.
  • Situações onde o custo de manutenção supera o benefício.

Abordagens de automação

Unitários

Testam pequenas unidades de código isoladamente. Frameworks: JUnit, NUnit, Jest, PyTest.

Integração

Testam comunicação entre módulos ou serviços. Focam em APIs e bancos de dados.

API

Validam endpoints, contratos e respostas. Ferramentas: Postman, RestAssured, Axios.

UI/E2E

Simulam jornada completa do usuário. Frameworks: Selenium, Cypress, Playwright.

Performance

Avaliam carga, stress e escalabilidade. Ferramentas: JMeter, LoadRunner, k6.

Contrato

Validam contratos entre serviços. Frameworks: Pact, Spring Cloud Contract.

Estratégia de automação

  • Comece pequeno: Automatize testes críticos e repetitivos primeiro.
  • Escolha ferramentas adequadas: Considere stack tecnológico e expertise da equipe.
  • Mantenha código limpo: Aplique boas práticas de desenvolvimento.
  • Page Objects/Modelos: Para testes UI, crie abstrações reutilizáveis.
  • Dados de teste: Use fixtures, factories ou geração dinâmica.
  • Relatórios: Integre com ferramentas de CI para dashboards.

Frameworks populares

Selenium WebDriver

Padrão para automação web. Suporta múltiplas linguagens.

Cypress

Rápido e confiável para aplicações modernas. JavaScript nativo.

Playwright

Multi-browser, multi-contexto. Suporte nativo a APIs modernas.

Appium

Automação mobile cross-platform para iOS e Android.

Exemplo prático: Teste automatizado com Selenium

Teste de Login - Java + Selenium

// Exemplo de teste automatizado
@Test
public void testLoginValido() {
    driver.get("https://minhaapp.com/login");

    WebElement emailField = driver.findElement(By.id("email"));
    WebElement passwordField = driver.findElement(By.id("password"));
    WebElement loginButton = driver.findElement(By.id("login-btn"));

    emailField.sendKeys("usuario@teste.com");
    passwordField.sendKeys("senha123");
    loginButton.click();

    WebElement dashboard = wait.until(ExpectedConditions
        .visibilityOfElementLocated(By.className("dashboard")));
    Assert.assertTrue(dashboard.isDisplayed());
}

Exemplo: Page Object Model

LoginPage.java

public class LoginPage {
    private WebDriver driver;

    @FindBy(id = "email")
    private WebElement emailField;

    @FindBy(id = "password")
    private WebElement passwordField;

    @FindBy(id = "login-btn")
    private WebElement loginButton;

    public LoginPage(WebDriver driver) {
        this.driver = driver;
        PageFactory.initElements(driver, this);
    }

    public DashboardPage login(String email, String password) {
        emailField.sendKeys(email);
        passwordField.sendKeys(password);
        loginButton.click();
        return new DashboardPage(driver);
    }
}
