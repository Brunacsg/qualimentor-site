Por que automatizar testes?

  • Redução significativa de tempo em testes repetitivos e regressão.
  • Maior confiabilidade e consistência na execução.
  • Execução contínua em pipelines de CI/CD.
  • Validação rápida de integrações críticas.
  • Liberação de profissionais de Quality Assurance para testes exploratórios e de valor agregado.
  • Feedback mais rápido para desenvolvedores.
  • Redução de custos a longo prazo.

Quando NÃO automatizar

  • Testes que serão executados apenas uma vez.
  • Testes com requisitos voláteis e mudanças frequentes.
  • Testes exploratórios que dependem de criatividade humana.
  • Testes de usabilidade que envolvem opinião subjetiva.
  • Situações onde o custo de manutenção supera o benefício.

Abordagens de automação

Unitários

Testam pequenas unidades de código isoladamente. Frameworks: JUnit, NUnit, Jest, PyTest.

Integração

Testam comunicação entre módulos ou serviços. Focam em APIs e bancos de dados.

API

Validam endpoints, contratos e respostas. Ferramentas: Postman, RestAssured, Axios.

UI/E2E

Simulam jornada completa do usuário. Frameworks: Cypress, Playwright e WebdriverIO com foco em JavaScript.

Performance

Avaliam carga, stress e escalabilidade. Ferramentas: JMeter, LoadRunner, k6.

Contrato

Validam contratos entre serviços. Frameworks: Pact, Spring Cloud Contract.

Estratégia de automação

  • Comece pequeno: Automatize testes críticos e repetitivos primeiro.
  • Escolha ferramentas adequadas: Considere stack tecnológico e expertise da equipe.
  • Mantenha código limpo: Aplique boas práticas de desenvolvimento.
  • Page Objects/Modelos: Para testes UI, crie abstrações reutilizáveis.
  • Dados de teste: Use fixtures, factories ou geração dinâmica.
  • Relatórios: Integre com ferramentas de CI para dashboards.

Frameworks populares

Cypress + JavaScript

Alta produtividade para testes E2E, componentes e APIs com excelente experiência de debugging.

Playwright

Boa opção para multi-browser e cenários cross-context, também com suporte forte a JavaScript.

Cypress Cloud

Executa paralelismo, gravações e analytics para escalar suites de automação em CI/CD.

Appium

Automação mobile cross-platform para iOS e Android.

Exemplo prático: Teste automatizado com Cypress

Teste de Login - Cypress + JavaScript

// cypress/e2e/login.cy.js
describe('Login', () => {
  it('deve autenticar com credenciais válidas', () => {
    cy.visit('https://minhaapp.com/login');

    cy.get('[data-cy="login-email-input"]').type('usuario@teste.com');
    cy.get('[data-cy="login-password-input"]').type('senha123');
    cy.get('[data-cy="login-submit-button"]').click();

    cy.url().should('include', '/dashboard');
    cy.get('[data-cy="dashboard-overview-section"]').should('be.visible');
  });
});

Exemplo: Page Object Model

loginPage.js

export class LoginPage {
  visit() {
    cy.visit('/login');
  }

  fillEmail(email) {
    cy.get('[data-cy="login-email-input"]').clear().type(email);
  }

  fillPassword(password) {
    cy.get('[data-cy="login-password-input"]').clear().type(password);
  }

  submit() {
    cy.get('[data-cy="login-submit-button"]').click();
  }

  login(email, password) {
    this.visit();
    this.fillEmail(email);
    this.fillPassword(password);
    this.submit();
  }
}

Exemplo: Teste de API com Cypress

Teste de API - JavaScript

it('deve buscar um usuário existente', () => {
  cy.request({
    method: 'GET',
    url: '/api/users/123',
    headers: {
      Authorization: `Bearer ${Cypress.env('token')}`
    }
  }).then((response) => {
    expect(response.status).to.equal(200);
    expect(response.body.name).to.equal('João Silva');
    expect(response.body.email).to.equal('joao@teste.com');
  });
});
